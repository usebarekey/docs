## What you need

- a Barekey account
- an organization selected in the web app
- a project and stage you can access
- Node.js or Bun for local SDK examples

## 1. Log in with the CLI

```bash
barekey auth login
```

The CLI starts a device flow against `https://api.barekey.dev` by default.
It opens a browser window and asks you to approve the session for your active organization.

Verify the local session:

```bash
barekey auth whoami
```

## 2. Set your local target

Create a `barekey.json` file in your app directory:

```json
{
  "org": "acme-1234",
  "project": "web-4567",
  "environment": "development"
}
```

The CLI walks up the current directory tree until it finds this file.
You can still override any value with `--org`, `--project`, or `--stage`.

## 3. Create a variable

```bash
barekey env new DATABASE_URL "postgres://localhost:5432/app"
```

Create an `ab_roll` variable:

```bash
barekey env new NEW_CHECKOUT off --ab on --chance 0.2
```

Inspect the stage:

```bash
barekey env list
```

## 4. Read a variable from code

Initialize the SDK once:

```ts
import { BarekeyClient } from "@barekey/sdk";

const env = new BarekeyClient({
  baseUrl: "https://api.barekey.dev",
  projectSlug: "web-4567",
  stageSlug: "development",
  auth: {
    async getAccessToken() {
      return process.env.BAREKEY_ACCESS_TOKEN ?? "";
    },
  },
});

const databaseUrl = await env.get("DATABASE_URL").string();
```

Read an `ab_roll` variable deterministically:

```ts
const checkoutVariant = await env
  .get("NEW_CHECKOUT", {
    seed: user.id,
    key: user.id,
    dynamic: { ttl: 30_000 },
  })
  .string();
```

## 5. Pull resolved values into a file

```bash
barekey env pull --format dotenv --out .env.local
```

This resolves each variable for the selected stage and writes a dotenv file.
For `ab_roll`, use `--seed` and `--key` when you need deterministic output.
