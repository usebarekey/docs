Barekey currently documents these shipped kinds:

- `secret`
- `ab_roll`

## `secret`

A `secret` is a single stored value.

Create one:

```bash
barekey env new DATABASE_URL "postgres://localhost:5432/app" --type string
```

Read it with the CLI:

```bash
barekey env get DATABASE_URL
```

Read it with the SDK:

```ts
const databaseUrl = await barekey.get("DATABASE_URL");
```

## `ab_roll`

An `ab_roll` stores two values:

- A
- B

And a `chance` between `0` and `1`.

Create one:

```bash
barekey env new CHECKOUT_FLOW control --ab treatment --chance 0.2
```

That means value A is returned 20% of the time and value B is returned 80% of the time.

### Deterministic reads

Pass `seed` and `key` to keep the result stable for a given user or cohort.

CLI:

```bash
barekey env get CHECKOUT_FLOW --seed user-42 --key checkout-v1
```

SDK:

```ts
const variant = await barekey.get("CHECKOUT_FLOW", {
  seed: user.id,
  key: "checkout-v1",
});
```

### Inspecting the decision

Use `inspect()` in the SDK when you want the decision metadata:

```ts
const result = await barekey.get("CHECKOUT_FLOW", {
  seed: user.id,
  key: "checkout-v1",
}).inspect();

result.value;
result.decision;
result.selectedArm;
```

## Choosing between them

Use `secret` when you want one value for the whole stage.

Use `ab_roll` when you want one of two values, optionally pinned to a stable seed and key.
